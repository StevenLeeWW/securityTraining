Security Testing Lab

Exercise 1: Using OWASP ZAP for Vulnerability Scanning

Tasks:

  1. Access OWASP ZAP at http://localhost:8090
  2. Perform a baseline scan of the vulnerable application
  3. Document all alerts generated by ZAP
  4. Categorize the findings by severity
  5. Verify at least three of the findings manually to confirm they are real vulnerabilities

Documentation Template:

# ZAP Scanning Results ## High Severity Findings - [Finding 1] - Description: - URL: - Manual Verification Steps: - Remediation: ## Medium Severity Findings - [Finding 1] - Description: - URL: - Manual Verification Steps: - Remediation: ## Low Severity Findings - [Finding 1] - Description: - URL: - Manual Verification Steps: - Remediation:

Hint: Focus on authentication, sessions, and input validation findings

Exercise 2: Manual Security Testing

Tasks:

  1. Perform manual security testing on these areas:
    • Authentication bypass
    • SQL injection
    • Cross-site scripting
    • Insecure direct object references
    • Business logic flaws
  2. Document your testing approach, tools used, and findings
  3. Create a simple proof-of-concept for each vulnerability found

Documentation Template:

# Manual Security Testing# Manual Security Testing ## Authentication Testing - Test Name: - Steps Performed: - Tools Used: - Findings: - Proof of Concept: - Remediation: ## SQL Injection Testing - Test Name: - Steps Performed: - Tools Used: - Findings: - Proof of Concept: - Remediation: ## XSS Testing - Test Name: - Steps Performed: - Tools Used: - Findings: - Proof of Concept: - Remediation: ## IDOR Testing - Test Name: - Steps Performed: - Tools Used: - Findings: - Proof of Concept: - Remediation: ## Business Logic Testing - Test Name: - Steps Performed: - Tools Used: - Findings: - Proof of Concept: - Remediation:

Hint: Use browser developer tools, Burp Suite or Postman to capture and modify HTTP requests
