Security Headers Analysis Worksheet

Instructions: Analyze the security headers of the following websites using browser developer tools:

  1. Our vulnerable application (http://localhost:8080)
  2. A popular secure website (e.g., https://www.google.com)
Security Header Vulnerable App Secure Website Purpose
Content-Security-Policy Controls resources the browser is allowed to load
X-Frame-Options Prevents clickjacking attacks
X-XSS-Protection Enables browser's XSS filtering
X-Content-Type-Options Prevents MIME type sniffing
Strict-Transport-Security Forces HTTPS connections

Questions:

  1. Which headers are missing from our vulnerable application?
  2. What risks does each missing header create?
  3. How would you implement these headers in an ASP.NET application?
